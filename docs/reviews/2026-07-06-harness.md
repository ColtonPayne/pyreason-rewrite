**Run:** independent adversarial review per the charter's review gate — two Opus
reviewers, no shared context, each writing findings blind to the other
([reviewer A](2026-07-06-harness-reviewer-A.md) · [reviewer B](2026-07-06-harness-reviewer-B.md)).
Scope: `harness/` (compare, capture, run, seed case) + its three test modules, at commit

**Threat-model calibration:** single operator, own laptop, no privilege boundary — every
finding is rated purely on verdict correctness and harness reliability (a false PASS being
the worst failure), never on exposure. Both reviewers applied this framing.

**Headline:** each reviewer found one High the other missed — A: a stale artifact plus an
ignored capture return code could launder a native engine crash into a false PASS; B: the
`interpretation_dict` probe called `get_interpretation_dict()`, a method that does not
exist at the pin (only `get_dict()` does). Both Highs were verified firsthand against the
source before fixing; that verification also located where the wrong name came from — the
pin's own `docs/hello-world.py`, a stale doc that likewise carries a pre-pin `Fact`
signature (consolidator's finding, not from either raw review). Independence earned its
cost.
