# Review report — multi-step capture, allow_raise, and the resume/reset corpus

Scope: the capture step-list extension (`50dcaad`), the nine resume/restart/reset
cases (`bc368fb`), and the board/run commit (`5dfeef8`). One focused Opus reviewer
with no shared session context — the umbrella's scaled-down form for a contained
spine extension (same deviation as the session-3 gate; recorded in the ledger).
Raw reviewer output: [2026-07-06-steps-raw.md](2026-07-06-steps-raw.md).

The reviewer confirmed the wrapping design sound (step outcomes and allow_raise
records are per-id and shape-distinct, so a wrapped record can never compare
against a bare value) and the reset-family `get_time` split source-accurate at
the pin. Findings and their resolutions:

## Fixed

- **H1 (High, latent)** — a probe-less successful `reason` step banks only
  `{"raised": false}`, so a case shape existed where two engines could derive
  different interpretations and still pass. None of the nine triggered it (every
  successful reason step was observed); the hole is now closed in validation: a
  `reason` step must carry probes or declare `outcome_only: true`, making an
  unobserved reason an explicit authoring choice. The four reset-family cases
  whose trailing reason steps are expected-raise observations carry the marker.
- **M1 (Medium)** — a typo'd `reason` kwarg (e.g. `restrat`) would bank the
  engine's `TypeError` as behavior and pass self-proof while testing nothing.
  Reason args are now validated against the pin's keyword surface (`REASON_ARGS`)
  in both case forms — guard parity with `apply_settings`' unknown-knob refusal.
  `queries` joins the set when the harness can construct Query objects.
- **M3 (Medium)** — `reset-settings-restore`'s second run was fact-less (the
  first run consumes the fact globals), so its claimed rule-name trace rows never
  existed and the atom_trace restore was observed only through column shape. The
  case now re-adds the seed fact after `reset_settings`, so the second trace has
  firing rows without Clause columns and the purpose's claim is earned.
- **L1 (Low)** — `allow_raise` on an `expect_raise` probe would defeat that
  probe's missing-constructor guard (a binding fault banked as engine behavior).
  Now a validation fault; `expect_raise` records raises itself. The narrower
  residual — a missing engine binding on an ordinary `allow_raise` probe — is
  accepted: it would surface as a blatant cross-engine mismatch, not a pass.
- **L2 (Low)** — `reason-again-no-program`'s purpose claimed its outputs "match a
  fresh run," which the harness never computes. Reworded to what is proven: the
  again-first path completes via the shared `_reason` branch; the equality to an
  again-less run is read off the source, not measured.
- **L3 (Low)** — the unknown-op test asserted `"rest" in fault`, satisfied by
  `reset*` in the ops list; now asserts the quoted token `'rest'`.

All fixes regression-tested (fast tier 55 passed); the corpus re-ran green after
the case edits and the re-banked run is the session's verdict-of-record.

## Deferred with rationale

- **M2 (Medium)** — a raise on a probe *without* `allow_raise` exits the capture
  as a failure, which the runner judges `error` (harness/environment), not
  `divergent` — so a candidate engine that newly raises on an undeclared probe
  would wear the wrong label. Deferred: in self-proof mode (the only mode that
  runs today) both pairs share one engine, so a cross-engine raise asymmetry
  cannot occur; the routing decision (tag probe-raise failures in the artifact
  and judge them divergent) is queued for when oracle-vs-rewrite runs begin,
  alongside session 3's deferred cross-engine exception-message policy. The
  capture docstring now states the consequence so case authors declare
  `allow_raise` wherever a candidate engine could plausibly raise.
