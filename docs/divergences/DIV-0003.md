# DIV-0003 — fp + infer_edges re-derivation: the pin dies with a payload-less KeyError, the rewrite with a stable one

- id: DIV-0003
- case ids: none — the arm cannot bank under the committed comparison
  policy: both engines raise the same type at the same seam but with
  different message text *by adjudicated design* (below), and the
  harness's only per-probe policy form is a numeric tolerance
  (harness/compare.py `compare_probes`), which cannot cover a raise-record
  message difference. A steps-form case would therefore always score
  `divergent` on the reason step's outcome record. Per DIV-0002's
  precedent (its arm is likewise case-less), the record plus the two
  reproducer tests carry the evidence. Nearest committed case:
  `rule-from-csv-basic` — the corpus's only infer_edges mid-run edge
  addition, on the DEFAULT schedule, where the per-run world dict carries
  every existing edge and the arm is unreachable.
- surface items: setting:fp_version, fn:add_rule (infer_edges=True),
  fn:reason
- observed (input: `fp_version=True`; rule
  `Linked(X, Y) <- p(X), q(Y)` with `infer_edges=True`; facts
  `p(A):[1,1]`, `q(B):[1,1]` active [0,2]; `reason(timesteps=2)` — the
  rule infers edge (A,B) at one timestep and re-derives it at the next):
  - **oracle (pin e1a94af33e1f):** the fp schedule keeps per-timestep
    world dicts that never carry existing edges forward
    (interpretation_fp.py:144-184: only t=0 exists at init, and it is
    empty), so the re-derivation's `_add_edge` existing-edge branch
    (interpretation_fp.py:2244, `l not in interpretations_edge[edge].world`)
    looks up a world the current timestep's typed dict does not hold —
    the numba typed-dict getitem raises the PAYLOAD-LESS
    `builtins.KeyError()` (dictobject.py:778): `exc.args == ()`,
    `str(exc) == ''`, traceback's final line bare `KeyError`. Screened
    2/2 fresh processes 2026-07-12 (session 29); first observed as
    slice-4 review finding F2 (probe `probe-fp-infer-edges`, session 19),
    carried as adjudication-batch item B17.
  - **rewrite (adjudicated behavior, implemented session 29):** the same
    seam — `_add_edge`'s existing-edge branch on the fp path
    (src/pyreason/_interpretation.py) — raises `builtins.KeyError` with
    the stable, honest message
    `edge ('A', 'B') already exists but has no world at timestep 1: the
    fp schedule's per-timestep world maps do not carry existing edges
    forward, so an inferred edge cannot be re-derived at a later
    timestep`. (KeyError's str() renders its argument's repr, so a
    raise-record `message` field shows the text wrapped in quotes; the
    pin's payload-less raise renders as the empty string — the two can
    never compare equal, which is why no case banks the arm.) As found
    (slice-4 review F2, pre-adjudication) the rewrite's plain-dict lookup
    raised the key-bearing `KeyError(('A', 'B'))` — same type, same seam,
    a message the pin's typed dict erases.
- first-seen session: 19 (slice-4 review F2); adjudicated direction
  recorded session 25; implemented and filed session 29.
- classification: intentional-divergence — both engines crash at the same
  logical seam with the same exception type and the same outcome
  (reason() raises, nothing banks); only the message text differs, and it
  differs because the operator adjudicated honesty over imitating a numba
  internal (matching the pin byte-for-byte would mean raising a bare
  `KeyError()` at a guessed site to mimic the typed-dict getitem — exactly
  what the adjudication rejected).
- reproducer: tests/test_div_0003_reproducer.py (e2e-marked: subprocess
  run of the pinned engine on the fp+infer_edges re-derivation program,

  test_fp_infer_edges_rederivation_raises_the_adjudicated_stable_keyerror
  (fast tier: the adjudicated guarded raise, exact message).
- status: adjudicated
- adjudicated behavior: at `_add_edge`'s existing-edge branch, when the fp
  schedule's current-timestep world map lacks an edge that already exists
  in the edge list, raise `builtins.KeyError` with the stable message
  above — built only from the edge pair and the timestep, both
  deterministic — stating what actually happened (the edge exists; the fp
  per-timestep worlds do not carry it forward; the re-derivation cannot be
  applied). It deliberately does NOT imitate the pin's payload-less shape,
  which is a numba typed-dict internal, not a stated contract. The guard
  is gated to the fp path: on the default schedule every existing edge
  holds a world entry by construction and the branch is unreachable
  (`rule-from-csv-basic` PASS is the covering evidence for that path).
