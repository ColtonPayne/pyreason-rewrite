# DIV-0001 — query-filter recursion: the pin dies, the rewrite terminates

- id: DIV-0001
- case ids: none — the input is un-caseable (the pinned process dies before
  any artifact exists to compare; the harness taxonomy refuses to bank it,
  ledger session 14). Nearest committed cases: reason-queries-filter,
  reason-queries-no-match, reason-queries-no-match-edge-heavy (all acyclic —
  on every acyclic ruleset the two engines return the identical
  reachable-rule set, so no committed case can observe the difference).
- surface items: type:Query, fn:reason
- observed:
  - **oracle (pin e1a94af33e1f):** a query matching a self-recursive rule's
    head drives filter_ruleset's unguarded recursion through the clause
    targets until the process dies (screened session 14: exit 139/SIGSEGV
    before Python's RecursionError; re-screened session 21, slice-6 review —
    nonzero process death, no output artifact).
  - **rewrite:** `filter_ruleset` expands each predicate at most once
    (`seen`), terminates on every cyclic ruleset, and returns the identical
    reachable-rule set on every acyclic one
    (src/pyreason/_program.py::filter_ruleset).
- first-seen session: 14 (pin behavior screened + board-noted); divergence
  introduced session 21 (phase-3 slice 6, executing the session-14 idea
  seed "bound or guard the query-filter recursion").
- classification: accepted intentional divergence — documented, tested
  intentional behavior per AC-6 (adjudicated 2026-07-11; proposed as
  intentional-divergence by the slice-6 review, session 21)
- reproducer: tests/test_div_0001_reproducer.py (e2e-marked: subprocess run
  of the pinned engine on the self-recursive-match input, asserts the
  process dies) + tests/test_rewrite_output_surface.py::
  test_filter_ruleset_terminates_on_self_recursive_rule (the rewrite's
  terminating behavior at the same seam).
- status: adjudicated
- provisional behavior: the rewrite keeps the guarded expansion (terminate,
  identical reachable set on acyclic inputs — all any committed case
  exercises). The pinned de-duplication through `list(set(...))` is kept,

- verdict: **option (a), accepted by the operator 2026-07-11 (adjudicating
