# hivemind-copy — snapshots of the governing hivemind docs

Dated, verbatim snapshots (2026-07-15, operator-requested) of the out-of-repo documents
this campaign runs under, committed so the GitHub mirror is readable without a local
ai-hivemind checkout. **None of these is the source of truth**: the canonical files live
in the hivemind and govern; campaign sessions read those, edits land there, and these
copies are refreshed manually when it matters.

- [charter.md][pyreason-rewrite-hivemind-copy-charter] — the governing charter
  (canonical: `$HIVEMIND_ROOT/idea-synthesis/plans/hivemind-portability.md`): acceptance
  criteria, permission charter, session protocol.
- [agent-playbook.md][pyreason-rewrite-hivemind-copy-agent-playbook] — the agent
  constitution (canonical: `$HIVEMIND_ROOT/docs/agent-playbook.md`).
- [ai-coding-pitfalls.md][pyreason-rewrite-hivemind-copy-ai-coding-pitfalls] — the
  recurring-mistake catalogue (canonical: `$HIVEMIND_ROOT/docs/ai-coding-pitfalls.md`).
- [cross-repo-links.md][pyreason-rewrite-hivemind-copy-cross-repo-links] — the doc-code
  link convention this repo's markdown follows (canonical:
  `$HIVEMIND_ROOT/docs/cross-repo-links.md`).
- [security-model.md][pyreason-rewrite-hivemind-copy-security-model] — the deployment
  threat model reviews calibrate against (canonical:
  `$HIVEMIND_ROOT/docs/security-model.md`).

Cross-repo reference links inside the snapshots resolve only beside a local hivemind
checkout; on GitHub they 404 by design — the linked docs are not part of this repo.
